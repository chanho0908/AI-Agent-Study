### 1. **N+1 쿼리 문제 - `getAllUsersWithOrders()`**

**[영향도: 높음]**

```
문제: users 테이블에서 1번 쿼리 + 각 user마다 orders 쿼리
- 유저 100명 = 101번의 DB 쿼리 (1 + 100)
- 각 쿼리 마다 I/O 오버헤드 누적
- UI 블로킹으로 인한 ANR 위험
```

**개선된 코드:**

```kotlin
// ✅ JOIN을 활용한 단일 쿼리 + 메모리에서 매핑
fun getAllUsersWithOrders(): List<UserWithOrders> {
    val query = """
        SELECT u.id, u.name, u.email, 
               o.id as order_id, o.user_id, o.amount
        FROM users u
        LEFT JOIN orders o ON u.id = o.user_id
    """.trimIndent()
    
    val cursor = db.rawQuery(query, null)
    val userMap = mutableMapOf<Int, UserWithOrders>()
    
    while (cursor.moveToNext()) {
        val userId = cursor.getInt(0)
        val user = User(
            id = userId,
            name = cursor.getString(1),
            email = cursor.getString(2)
        )
        
        val order = if (cursor.isNull(3)) {
            null
        } else {
            Order(
                id = cursor.getInt(3),
                userId = cursor.getInt(4),
                amount = cursor.getDouble(5)
            )
        }
        
        userMap.getOrPut(userId) { UserWithOrders(user, mutableListOf()) }
            .orders.let { if (order != null) it.add(order) }
    }
    cursor.close()
    return userMap.values.toList()
}

// ✅ Coroutine 활용 (UI 스레드 블로킹 방지)
suspend fun getAllUsersWithOrdersAsync(): List<UserWithOrders> = withContext(Dispatchers.IO) {
    getAllUsersWithOrders()
}
```

### 2. **0으로 나누기 런타임 예외 - `getUserStats()`**

**[영향도: 높음]**

```
문제: orders가 빈 리스트일 때 avgAmount = totalAmount / 0
- ArithmeticException 발생으로 앱 크래시
- 정상 사용자(주문 없음)의 통계 조회 시 발생
```

**개선된 코드:**

```kotlin
fun getUserStats(userId: Int): UserStats {
    val cursor = db.rawQuery(
        "SELECT COALESCE(SUM(amount), 0) as total, COUNT(*) as count FROM orders WHERE user_id = ?",
        arrayOf(userId.toString())
    )
    
    val stats = if (cursor.moveToFirst()) {
        val totalAmount = cursor.getDouble(0)
        val orderCount = cursor.getInt(1)
        
        // ✅ 0으로 나누기 방지
        val avgAmount = if (orderCount > 0) totalAmount / orderCount else 0.0
        
        UserStats(orderCount, totalAmount, avgAmount)
    } else {
        UserStats(0, 0.0, 0.0)
    }
    
    cursor.close()
    return stats
}
```

## 2. 즉시 수정 필요 TOP 3

### 🚨 #1 — SQL 인젝션 취약점 (3곳)
`findUserByName()`, `processUser()`, `getUserStats()` 모두 사용자 입력값을 SQL 쿼리에 직접 문자열로 삽입하고 있어, 공격자가 전체 DB를 열람·변조할 수 있는 **치명적 보안 취약점**이 존재함. `rawQuery()`의 두 번째 인자(`selectionArgs`)를 반드시 사용하거나 `ContentValues` + `db.update()`로 즉시 교체해야 함.

### 🚨 #2 — 하드코딩된 관리자 비밀번호 및 JWT 토큰
`ADMIN_PASSWORD = "admin1234!"`, `SECRET_TOKEN = "Bearer eyJ..."` 등의 민감 정보가 소스코드에 평문으로 박혀 있어 APK 역컴파일 즉시 노출됨. Git 히스토리에도 영구 기록되므로, **지금 당장 자격증명을 폐기·갱신**하고 Android Keystore 또는 서버사이드 발급 방식으로 전환해야 함.

### 🚨 #3 — 0으로 나누기 런타임 크래시 (`getUserStats()`)
주문이 없는 사용자(`orderCount = 0`)에 대해 `avgAmount = totalAmount / orderCount`를 수행하여 `ArithmeticException`으로 **앱이 즉시 크래시**됨. `if (orderCount > 0) totalAmount / orderCount else 0.0` 또는 DB 레벨에서 `CASE WHEN COUNT(*) > 0 THEN ... END`로 즉시 수정 필요.
