다음 순서로 작업을 수행하세요:

## 1단계: 계획 수립
- Task tool을 사용하여 subagent_type=Plan으로 에이전트를 호출
- 사용자의 요구사항을 분석하고 상세한 구현 계획을 수립
- 계획에는 다음이 포함되어야 합니다:
    * 변경할 파일 목록
    * 각 파일의 변경 내용
    * 구현 순서
    * 예상되는 영향 범위

## 2단계: 계획 제시 및 승인 대기
- Plan 에이전트가 수립한 계획을 사용자에게 명확하게 제시
- 사용자의 승인 또는 수정 요청을 기다림

## 3단계: 구현 (사용자 승인 후)
- 사용자가 계획을 승인하면:
    * Task tool을 사용하여 subagent_type=implementer로 에이전트를 호출
    * 1단계에서 수립한 계획을 implementer에게 상세히 전달
    * implementer가 계획에 따라 코드를 구현

## 중요 사항
- 각 단계는 순차적으로 진행되어야 합니다
- 2단계에서 사용자 승인 없이 3단계로 넘어가지 마세요
- Plan과 Implementer를 **별도의 Task 호출**로 실행하세요

사용자가 요구사항을 제공하면, 위 3단계를 순차적으로 수행하세요.
