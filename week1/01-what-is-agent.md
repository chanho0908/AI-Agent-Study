# AI Agent란 무엇인가

## 한 줄 정의

> AI Agent는 **목표를 주면 스스로 판단하고 행동하며 결과를 만들어내는 AI 시스템**이다.

---

## LLM과 AI Agent의 차이

일반적인 LLM(ChatGPT, Claude 등)은 이렇게 동작한다.

```
사용자 입력 → LLM → 텍스트 응답
```

한 번 질문하면 한 번 답한다. 끝.

AI Agent는 다르다.

```
목표 입력 → [생각 → 행동 → 관찰] → [생각 → 행동 → 관찰] → ... → 최종 결과
```

목표를 달성할 때까지 **반복적으로 생각하고, 행동하고, 결과를 관찰**한다.

### 예시로 비교

**LLM에게 물어볼 때:**
> "오늘 서울 날씨 알려줘"

LLM은 자신의 학습 데이터 기준으로 답한다. 실시간 정보를 모른다.

**Agent에게 시킬 때:**
> "오늘 서울 날씨 알려줘"

Agent는 이렇게 동작한다.
1. "날씨를 알려면 웹 검색이 필요하다" (생각)
2. 날씨 API 또는 웹 검색 실행 (행동)
3. 검색 결과 확인 (관찰)
4. 결과를 바탕으로 답변 생성 (최종 응답)

---

## 이미 Agent를 쓰고 있다: Claude Code

Claude Code는 AI Agent의 좋은 예시다.

"이 파일의 버그 고쳐줘"라고 하면:
1. 파일을 읽는다 (Read 도구 사용)
2. 문제를 파악한다 (생각)
3. 수정 방법을 결정한다 (계획)
4. 파일을 수정한다 (Edit 도구 사용)
5. 결과를 확인한다 (관찰)

단순히 "이렇게 고치면 됩니다"라고 텍스트만 반환하는 게 아니라, **직접 파일을 읽고 수정하는 행동**을 한다. 이게 Agent다.

---

## Agent를 정의하는 세 가지 특성

### 1. 자율성 (Autonomy)
외부의 지시 없이 스스로 다음 행동을 결정한다.

### 2. 도구 사용 (Tool Use)
텍스트 생성 이상의 행동을 한다. 검색, 파일 읽기/쓰기, API 호출, 코드 실행 등.

### 3. 반복 루프 (Agentic Loop)
목표를 달성할 때까지 "생각 → 행동 → 관찰"을 반복한다.

---

## Agent의 스펙트럼

Agent는 얼마나 자율적으로 동작하느냐에 따라 스펙트럼이 있다.

```
덜 자율적                                              더 자율적
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
단순 LLM   →   RAG   →   단일 Agent   →   멀티 Agent
```

- **단순 LLM**: 질문-답변만 함
- **RAG (Retrieval Augmented Generation)**: 외부 문서를 검색해서 참고하고 답함
- **단일 Agent**: 도구를 사용하며 목표를 달성
- **멀티 Agent**: 여러 Agent가 역할을 나눠서 협력

---

## 정리

| 구분 | LLM | AI Agent |
|------|-----|----------|
| 동작 방식 | 입력 → 출력 (1회) | 목표 → 반복 루프 → 결과 |
| 외부 도구 사용 | 불가 | 가능 (검색, 파일, API 등) |
| 자율성 | 없음 | 있음 |
| 실시간 정보 | 모름 | 도구로 얻을 수 있음 |
| 예시 | ChatGPT 기본 대화 | Claude Code, AutoGPT |
