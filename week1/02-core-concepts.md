# AI Agent 핵심 구성요소 및 패턴

## Agent의 4가지 구성요소

Agent는 보통 아래 4가지 요소로 구성된다.

```
┌─────────────────────────────────────┐
│              AI Agent               │
│                                     │
│  ┌──────────┐    ┌──────────────┐   │
│  │Perception│    │   Planning   │   │
│  │(인식)    │    │   (계획)     │   │
│  └──────────┘    └──────────────┘   │
│                                     │
│  ┌──────────┐    ┌──────────────┐   │
│  │  Memory  │    │    Action    │   │
│  │  (기억)  │    │   (행동)     │   │
│  └──────────┘    └──────────────┘   │
└─────────────────────────────────────┘
```

---

### 1. Perception (인식)

Agent가 외부 세계에서 정보를 받아들이는 부분.

- 사용자의 텍스트 입력
- 파일 내용
- 웹 검색 결과
- 이미지, 코드, 데이터 등

**Claude Code 예시:** 사용자가 "이 파일 읽어봐"라고 하면, Read 도구로 파일 내용을 읽는 것이 Perception이다.

---

### 2. Memory (기억)

Agent가 정보를 저장하고 나중에 꺼내 쓰는 부분.

| 종류 | 설명 | 예시 |
|------|------|------|
| **단기 기억 (In-context)** | 현재 대화 창에 있는 내용 | 이번 대화의 맥락 |
| **장기 기억 (External)** | 대화 밖에 저장된 정보 | 파일, DB, 벡터 DB |
| **에피소드 기억** | 과거에 했던 일의 기록 | 이전 실행 로그 |

**Claude Code 예시:** CLAUDE.md 파일이나 memory 폴더가 장기 기억 역할을 한다. Claude Code를 새로 시작해도 이 파일들을 읽어서 이전 컨텍스트를 유지할 수 있다.

---

### 3. Planning (계획)

목표를 달성하기 위해 어떤 순서로 무엇을 할지 결정하는 부분.

두 가지 방식이 있다.

**① 사전 계획 (Plan-then-Execute)**
```
목표 → 전체 계획 수립 → 계획대로 실행
```
장점: 체계적. 단점: 중간에 상황이 바뀌면 유연하지 못함.

**② 반응형 계획 (ReAct)**
```
목표 → 생각 → 행동 → 관찰 → 생각 → 행동 → ...
```
생각하면서 동시에 행동하고, 관찰한 결과를 바탕으로 다음 행동을 결정한다.

> ReAct = **Re**asoning + **Act**ing

대부분의 현대 Agent는 ReAct 패턴을 따른다.

---

### 4. Action (행동)

Agent가 실제로 외부 세계에 영향을 미치는 부분. **Tool Use**라고도 한다.

| 행동 종류 | 예시 |
|----------|------|
| **정보 검색** | 웹 검색, 파일 읽기, DB 조회 |
| **파일 조작** | 파일 생성, 수정, 삭제 |
| **코드 실행** | Python 코드 실행, 터미널 명령 |
| **외부 API 호출** | 날씨 API, 슬랙 메시지 전송 |
| **다른 Agent 호출** | 서브 Agent에게 작업 위임 |

---

## 핵심 패턴: ReAct

ReAct는 현재 가장 널리 사용되는 Agent 패턴이다.

```
Thought: 나는 날씨 정보가 필요하다. 웹 검색을 써야겠다.
Action: search("서울 오늘 날씨")
Observation: 서울 현재 기온 18도, 맑음
Thought: 날씨 정보를 얻었다. 이제 답변을 만들 수 있다.
Action: 최종 답변 생성
```

**Claude Code가 이 패턴을 따른다.** 코드를 수정하기 전에 파일을 먼저 읽고(Action), 내용을 확인하고(Observation), 어떻게 수정할지 판단하고(Thought), 수정한다(Action).

## Tool Use (도구 사용)

Tool Use는 LLM이 외부 도구를 호출할 수 있게 해주는 기능이다.

작동 방식:
1. 개발자가 "이런 도구가 있습니다"라고 LLM에게 알려준다
2. LLM이 필요할 때 "이 도구를 이렇게 호출해주세요"라고 요청한다
3. 코드가 실제로 도구를 실행하고 결과를 LLM에 돌려준다
4. LLM이 결과를 보고 다음 행동을 결정한다

```python
# 이런 구조로 동작 (개념적 예시)
tools = [
    {
        "name": "search_web",
        "description": "웹에서 정보를 검색합니다",
        "parameters": {"query": "검색어"}
    }
]

# LLM이 판단해서 도구 호출을 요청
response = llm.call(
    message="서울 날씨 알려줘",
    tools=tools
)
# → LLM: "search_web 도구로 '서울 오늘 날씨'를 검색해주세요"

# 코드가 실제 검색 실행 후 결과를 LLM에 전달
result = search_web("서울 오늘 날씨")
# → LLM이 결과를 보고 최종 답변 생성
```

## Prompt Engineering과의 관계

단순 LLM을 쓸 때도 프롬프트가 중요하지만, Agent에서는 더 중요하다.

Agent의 프롬프트는 보통 이런 역할을 한다.

- **시스템 프롬프트**: Agent의 역할, 사용 가능한 도구, 행동 원칙을 정의
- **사용자 입력**: 달성해야 할 목표
- **도구 결과**: 도구 실행 결과 (Observation)

프롬프트를 잘 설계할수록 Agent가 더 정확하게 판단하고 행동한다.

---

## 정리

```
목표 입력 (Perception)
    ↓
어떻게 달성할까? (Planning - Thought)
    ↓
도구 실행 (Action - Tool Use)
    ↓
결과 확인 (Observation)
    ↓
목표 달성? → NO → 다시 Planning으로
    ↓ YES
최종 결과 반환
```
